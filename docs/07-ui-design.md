# UI 设计文档 — 学生"一站式"自主管理过程管理系统

> 文档版本：v2.0 | 创建日期：2026-06-06 | 输入来源：[02-prd.md](./02-prd.md)
> 设计寄存器：Product（设计服务于产品，非品牌展示型）

---

## 1. 设计定位

### 1.1 场景句

高校辅导员在办公室日光灯下用 PC 审批入团申请，大二学生在课间用手机查看勤工助学岗位，社团负责人在图书馆用平板提交活动策划。系统需同时服务管理效率与操作便捷，PC 端承载密集数据操作，移动端保证核心流程可用。

### 1.2 设计目标

1. **一站式入口**：学生从统一工作台进入四大业务，无需跑多个办公室。
2. **流程可视化**：申请、审批、报名、签到、录用、考勤等状态通过色彩标签和时间线清晰追踪。
3. **角色差异化**：不同角色看到不同菜单、不同 Dashboard 视图和操作按钮，后端 `user.menus` 动态渲染。
4. **响应式兼容**：PC 端优先保证管理效率，移动端保证核心操作可用。
5. **数据驱动管理**：Dashboard 突出统计概览、趋势图表和快捷操作入口。

### 1.3 色彩策略

**克制（Restrained）**：产品型工具界面，品牌色仅用于主操作和当前选中态，中性色承载大部分表面。不追求视觉冲击，追求操作可信度和信息密度。

## 2. 设计系统

### 2.1 色彩

基于 OKLCH 色彩空间定义，确保感知均匀和对比度达标。所有色值提供 HEX 降级用于 Element Plus 兼容。

#### 品牌色与表面色

| Token | OKLCH | HEX | 用途 |
|-------|-------|-----|------|
| `--color-brand` | oklch(0.45 0.18 260) | `#1d4ed8` | 侧边栏品牌区、主按钮强调色 |
| `--color-brand-hover` | oklch(0.50 0.18 260) | `#2563eb` | 品牌色悬停态 |
| `--color-brand-subtle` | oklch(0.95 0.02 260) | `#eef2ff` | 品牌色浅底（选中行背景等） |
| `--color-surface` | oklch(0.98 0.005 260) | `#f5f7fb` | 页面底色 |
| `--color-surface-raised` | oklch(1.00 0 0) | `#ffffff` | 卡片/侧边栏/弹窗底色 |
| `--color-surface-sunken` | oklch(0.96 0.005 260) | `#f0f1f5` | 筛选区/分区卡片头底色 |
| `--color-ink` | oklch(0.20 0.02 260) | `#1f2937` | 主文字 |
| `--color-ink-muted` | oklch(0.45 0.01 260) | `#6b7280` | 辅助文字（≥4.5:1 对比度） |
| `--color-ink-faint` | oklch(0.65 0.01 260) | `#9ca3af` | 占位文字（仅用于 placeholder，不可用于正文） |
| `--color-border` | oklch(0.88 0.01 260) | `#e5e7eb` | 默认边框 |
| `--color-border-strong` | oklch(0.75 0.01 260) | `#d1d5db` | 强调边框（表格等） |

#### 语义色（与 Element Plus `el-tag` type 对齐）

| 语义 | Token | HEX | el-tag type | 使用场景 |
|------|-------|-----|-------------|----------|
| 成功/通过 | `--color-success` | `#67C23A` | success | 已完成、签到成功、活跃、已录用、已发放、已核实 |
| 警告/待处理 | `--color-warning` | `#E6A23C` | warning | 待审批、待处理、预警、待核实、待发放、已报名 |
| 危险/异常 | `--color-danger` | `#F56C6C` | danger | 驳回、异常、缺勤、已拒绝、已解散 |
| 信息/中性 | `--color-info` | `#909399` | info | 草稿、未开始、停用、已关闭、已归档、已取消 |

#### 组件主色

Element Plus 默认蓝 `#409EFF` 保持不变，用于组件库内置交互态。品牌色 `#1d4ed8` 仅用于侧边栏品牌区和自定义主按钮，避免与组件主色冲突。

### 2.2 字体

| 用途 | 字体栈 | 说明 |
|------|--------|------|
| 全局 | `Inter, "Microsoft YaHei", system-ui, sans-serif` | 一个字体族覆盖标题/正文/数据，通过字重和字号建立层级 |

**字号层级**（固定 rem，不使用 clamp 流式缩放）：

| 层级 | 大小 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| h1 | 1.75rem (28px) | 700 | 1.3 | 页面标题 |
| h2 | 1.375rem (22px) | 600 | 1.35 | 分区标题 |
| h3 | 1.125rem (18px) | 600 | 1.4 | 卡片标题 |
| body | 0.875rem (14px) | 400 | 1.6 | 正文、表格内容 |
| caption | 0.75rem (12px) | 400 | 1.5 | 辅助说明、标签 |

> 层级比 1.27 → 1.22 → 1.17 → 1.0，保持紧凑但可辨的层级梯度。管理端允许 14px 正文以承载信息密度；学生端核心操作区可适当放大至 15-16px。

### 2.3 间距

基于 4px 基础单位的 8 倍数系统：

| Token | 值 | 用途 |
|-------|-----|------|
| `--space-1` | 4px | 图标与文字间距 |
| `--space-2` | 8px | 紧凑元素间距 |
| `--space-3` | 12px | 筛选区内边距 |
| `--space-4` | 16px | 页面容器 gap、卡片内边距 |
| `--space-5` | 20px | 分区间距 |
| `--space-6` | 24px | 大分区间距 |
| `--space-8` | 32px | 页面顶部间距 |

### 2.4 圆角

| Token | 值 | 用途 |
|-------|-----|------|
| `--radius-sm` | 4px | 标签、小按钮 |
| `--radius-md` | 6px | 输入框、筛选区 |
| `--radius-lg` | 8px | 卡片、弹窗 |
| `--radius-full` | 9999px | 胶囊标签 |

### 2.5 阴影

| Token | 值 | 用途 |
|-------|-----|------|
| `--shadow-sm` | `0 1px 2px rgba(0,0,0,0.05)` | 卡片默认 |
| `--shadow-md` | `0 4px 8px rgba(0,0,0,0.08)` | 弹窗、悬浮层 |
| `--shadow-lg` | `0 8px 16px rgba(0,0,0,0.12)` | 下拉菜单 |

> 不在同一元素上同时使用 border + 宽模糊阴影（ghost-card 模式）。选择其一：实线边框或紧凑阴影。

### 2.6 z-index 层级

| 层级 | 值 | 用途 |
|------|-----|------|
| dropdown | 1000 | 下拉菜单 |
| sticky | 1100 | 固定表头/顶栏 |
| modal-backdrop | 2000 | 弹窗遮罩 |
| modal | 2100 | 弹窗 |
| toast | 3000 | 消息提示 |
| tooltip | 4000 | 工具提示 |

## 5. 导航结构

### 5.1 菜单渲染机制

- 后端登录接口返回 `user.menus` 数组，每项含 `key`、`title`、`path`、`icon` 字段。
- 前端 `MainLayout.vue` 通过 `v-for` 渲染 `el-menu-item`，使用 router 模式。
- 当前激活菜单通过 `activeMenu` 计算属性确定：优先精确匹配路径，否则取最长前缀匹配。
- 菜单可见性完全由后端 RBAC 控制，前端不做硬编码。

### 5.2 角色导航差异

| 路由路径 | 页面 | admin | teacher | student_cadre | student |

| `/` | 工作台 | 可见 | 可见 | 可见 | 可见 |
| `/services` | 服务入口 | 可见 | 可见 | 可见 | 可见 |
| `/members` | 团员发展 | 可见 | 可见 | 可见 | 不可见 |
| `/members/new` | 新增档案 | 可见 | 可见 | 可见 | 不可见 |
| `/members/:id/edit` | 编辑档案 | 可见 | 可见 | 可见 | 不可见 |
| `/members/:id` | 电子档案 | 可见 | 可见 | 可见 | 不可见 |

## 14. 交互规范

### 14.1 组件状态完整性

每个可交互组件必须覆盖以下状态：

| 状态 | 说明 | 实现方式 |
|------|------|----------|
| default | 默认态 | 基础样式 |
| hover | 悬停态 | `:hover` 样式变化 |
| focus | 聚焦态 | `:focus-visible` 轮廓，键盘可达 |
| active | 激活态 | `:active` 按压反馈 |
| disabled | 禁用态 | `:disabled` 灰色 + cursor: not-allowed |
| loading | 加载中 | `:loading` 属性，按钮旋转图标 |
| error | 错误态 | 红色边框 + 错误提示文字 |

### 14.2 动效规范

| 交互 | 时长 | 缓动 | 说明 |
|------|------|------|------|
| 按钮悬停/按压 | 150ms | ease-out | 颜色/阴影变化 |
| 弹窗打开/关闭 | 200ms | ease-out-quart | 淡入+缩放 |
| 侧边栏展开/折叠 | 250ms | ease-out-quart | 宽度过渡 |
| 页面切换 | 200ms | ease-out | 淡入淡出 |
| 列表项入场 | 150ms | ease-out | 交错延迟 ≤ 50ms |

> 所有动效需 `@media (prefers-reduced-motion: reduce)` 降级为即时切换。

### 14.3 可访问性

- 正文文字不小于 14px，管理表格不小于 14px，学生端核心操作不小于 15px
- 所有按钮、表单、筛选项需支持键盘聚焦状态（`:focus-visible`）
- 状态不可只依赖颜色，应同时使用文字标签（`el-tag` 含文字）
- 移动端可点击区域不小于 44px
- 图标按钮需 `aria-label`
- 对比度：正文 ≥ 4.5:1，大文字 ≥ 3:1

## 16. 设计验收标准

1. 普通学生可从工作台看到个人统计和服务画像，通过服务入口页进入四大业务入口。
2. 管理端用户可在 Dashboard 看到统计概览、趋势图表和快捷管理入口。
3. 不同角色登录后菜单和操作按钮符合权限矩阵（路由守卫 `meta.roles` + 后端 `user.menus` 双重保障）。
4. 团员发展全流程（入团申请 → 积极分子 → 发展对象 → 政审备案）通过时间线可视化展示。
5. 社团活动全流程（创建 → 审批 → 报名 → 签到 → 总结归档）状态标签和操作按钮随状态变化。
6. 勤工助学全流程（岗位发布 → 报名 → 录用 → 考勤 → 薪资）在岗位详情页一站式管理。
7. 社区队伍管理（队伍 CRUD + 成员管理 + 值班排班 + 签到签退 + 服务画像）完整闭环。
8. 所有列表页均具备筛选、状态标签、分页能力。
9. 表单校验提示展示在字段附近，删除/关闭等危险操作需二次确认。
10. 文件上传通过 MinIO 存储，支持分类上传和在线访问。
